	We ran the command ls - l and it showed me the current permission granted to all files and directories these permissions are often in the format of drwxrwxrwx, read and understand what this means in quite simple so to start off the letter d stands for directory in some case it might just have a - this means that it is just a normal file not a directory, so now we are going to group in pair of three so it goes rwx,rwx,rwx this 3 pairs are the permission that are or have been granted to user,group, and other in that order so for user it has all three permissions which are read(for files this is the ability to read the contents of file and for directories this mean the ability to read everything in said directory such files and subdirectories) this permission is represented by the letter r, we also have the write permission(for files this permission grants the ability to make modifications on the file contents and for directories, this is the ability to create new files in the directory) this permission is represented through the w, and finally we have the execute permission(for files this permission grants the ability to execute the file if its a program and for directories this is the ability to enter the directory and access its files) this permission is ofter represented by x and if a permission was not granted you will see a - in the place of where the permission should be
	
	So after running ls -l command we ran the chmod 777 which granted the other to the permissions of read,write, and execute this is a security risk because the owner known as other is essentially all other users on said system and we dont want other users to be able to edit and execute my file or a file.

	So to fix this we ran the command chmod 644 which removed the write and execute permission only allowing the owner group others to only read the contents of the file
